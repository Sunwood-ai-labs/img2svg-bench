# コントリビューション

## 受け入れ対象

- 新規変換ランナーの追加
- 前処理の改善
- 指標の追加・拡張
- ドキュメント整備

## 新しいランナー追加手順

1. `scripts/` 配下にランナー実装を追加（既存スタイルに合わせる）
2. CLI から入力画像を受け取り、決定論的な出力を生成する
3. 実験集約処理にプリセット名とメタデータを追加
4. `output/` に成果物を配置し、レポートが取り込めるようにする

## ドキュメント運用ルール

- 英語 docs を先に更新
- 同内容の日本語ページを `docs/ja/` に追加
- UV ベースの Python 実行フローを保ったサンプルを維持

## レビュー前チェック

PR には以下を添付してください。

- 変更内容の要約
- 実行コマンドと、変更結果の確認ログ
- 既存出力互換性に対する影響（あれば）
