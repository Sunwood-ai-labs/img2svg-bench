# ベンチマーク設計

## 中核コンセプト

このリポジトリは、以下を明示することで比較性を保ちます。

1. 入力の扱い
2. 任意の前処理
3. 変換プリセット実行
4. 指標収集
5. レポート作成

これにより、目視だけでは追えない比較条件のズレを減らし、再現可能性を高めます。

## 1 回の実行とは

1 回の実行は次の組み合わせです。

- 元画像
- 前処理種別（未処理 / 背景除去）
- ランナー（現在は `VTracer`）
- プリセット名

## CSV と Markdown を併用する理由

CSV は自動処理に向く機械可読データを提供し、Markdown は人間が見て判断しやすい差分確認用の参照先を提供します。

## 拡張方針

新しいランナーを追加する場合は、同じ契約で接続します。

- CLI で入力画像を受け取る
- 出力先を決まったフォルダに書き出す
- 実行時間や構造指標を記録する
- 再現手順を示すメタ情報を残す

この契約が保たれていれば、既存比較フローと整合して評価できます。
